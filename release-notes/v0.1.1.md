# RazorConsole v0.1.1

**Released: November 16, 2025**

RazorConsole v0.1.1 expands the component catalog, freshens the documentation site, and tightens focus/input behavior while preparing the codebase for the next .NET wave.

## Highlights
- **New Components**: `Scrollable` and `SpectreCanvas` components unlock richer console layouts ([#114](https://github.com/LittleLittleCloud/RazorConsole/pull/114), [#78](https://github.com/LittleLittleCloud/RazorConsole/pull/78))
- **Modern Runtime Support**: .NET 10 target keeps the libraries ready for the upcoming runtime ([#113](https://github.com/LittleLittleCloud/RazorConsole/pull/113))
- **Improved Reliability**: Focus and text input fixes enhance keyboard-driven workflows ([#108](https://github.com/LittleLittleCloud/RazorConsole/pull/108), [#59](https://github.com/LittleLittleCloud/RazorConsole/pull/59))
- **Rich Examples**: Counter, Login Form, and Local File Explorer showcase real-world patterns ([#93](https://github.com/LittleLittleCloud/RazorConsole/pull/93), [#97](https://github.com/LittleLittleCloud/RazorConsole/pull/97), [#95](https://github.com/LittleLittleCloud/RazorConsole/pull/95))
- **Documentation Overhaul**: Complete website revamp with dark mode and professional branding ([#63](https://github.com/LittleLittleCloud/RazorConsole/pull/63), [#75](https://github.com/LittleLittleCloud/RazorConsole/pull/75))

## Detailed Changes

### Components & Rendering
- Added the `Scrollable` component and corresponding gallery demo to handle large render trees ([#114](https://github.com/LittleLittleCloud/RazorConsole/pull/114))
- Introduced `SpectreCanvas` for low-level drawing scenarios in Spectre.Console ([#78](https://github.com/LittleLittleCloud/RazorConsole/pull/78))
- Enabled the `start` attribute on ordered lists so Markdown content can continue numbering mid-document ([#116](https://github.com/LittleLittleCloud/RazorConsole/pull/116))
- Simplified the select component styling to better match Spectre expectations ([#98](https://github.com/LittleLittleCloud/RazorConsole/pull/98))

### Focus & Input Reliability
- Fixed `FocusManager` to automatically focus the next element when the current one disappears ([#108](https://github.com/LittleLittleCloud/RazorConsole/pull/108))
- Ensured `TextInput` keeps its value if a parent re-renders without invoking `ValueChanged` ([#59](https://github.com/LittleLittleCloud/RazorConsole/pull/59))

### Samples & Gallery
- Added a Counter sample application demonstrating component-driven state management ([#93](https://github.com/LittleLittleCloud/RazorConsole/pull/93))
- Added a CLI login form example that highlights validation and error states ([#97](https://github.com/LittleLittleCloud/RazorConsole/pull/97))
- Added a Local File Explorer example covering file I/O and dynamic table updates ([#95](https://github.com/LittleLittleCloud/RazorConsole/pull/95))
- Updated the README to introduce the new components and gallery additions ([#117](https://github.com/LittleLittleCloud/RazorConsole/pull/117))

### Tooling, Docs & Website
- Launched the documentation website built with Vite, TypeScript, TailwindCSS, and shadcn/ui ([#63](https://github.com/LittleLittleCloud/RazorConsole/pull/63))
- Added dark theme support with theme switcher using Zustand ([#75](https://github.com/LittleLittleCloud/RazorConsole/pull/75))
- Refreshed website branding with RazorConsole logo variants and assets ([#73](https://github.com/LittleLittleCloud/RazorConsole/pull/73), [#67](https://github.com/LittleLittleCloud/RazorConsole/pull/67))
- Fixed dark theme regressions and routing when hosting under a sub-path ([#83](https://github.com/LittleLittleCloud/RazorConsole/pull/83), [#68](https://github.com/LittleLittleCloud/RazorConsole/pull/68))
- Removed outdated focus management docs and refreshed the Discord invite ([#69](https://github.com/LittleLittleCloud/RazorConsole/pull/69), [#71](https://github.com/LittleLittleCloud/RazorConsole/pull/71))
- Added stricter `.editorconfig` formatting rules and contribution guidelines ([#80](https://github.com/LittleLittleCloud/RazorConsole/pull/80), [#61](https://github.com/LittleLittleCloud/RazorConsole/pull/61))

### Build & CI
- Added .NET 10 support across projects ([#113](https://github.com/LittleLittleCloud/RazorConsole/pull/113))
- Prevented NuGet publishing when no source code changes are present ([#66](https://github.com/LittleLittleCloud/RazorConsole/pull/66))
- Fixed artifact upload paths in workflows ([`34a44c7`](https://github.com/LittleLittleCloud/RazorConsole/commit/34a44c7))
- Updated the deploy-pages workflow to use the current Node.js setup action ([`a8cb696`](https://github.com/LittleLittleCloud/RazorConsole/commit/a8cb696))
